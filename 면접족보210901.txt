1. JVM 에 대하여 설명하시오.
Java Virtual Machine의 약자. 자바가상머신.
자바프로그램이 OS(운영체제)에서 실행될 수 있게 하는 소프트웨어.
OS의 종류에 상관없이 java를 실행하는 환경을 만들어 줌.
자바번역기. java코드가 작성되면 javac를 통해 class파일이 생성되고
jvm이 생성된 class 파일을 OS로 보내 java와 OS 사이의 중재자 역할을 함.
또 한정된 메모리를 최대한 효율적으로 관리하고 최적화하는 기능도 있음.

2. 일반적인 프로그램과 java 가 실행되는 차이점은?
일반적인 프로그램-특정 운영체제에 맞춰서 만들어지므로 프로그램을 바로 실행. (OS->RAM->CPU)
java프로그램-JVM이 OS로부터 필요로하는 메모리를 할당받은 후 관리.
javac(자바컴파일러)가 소스코드를 읽어서 java바이트 코드(.class)로 변환한 후 JVM으로 로딩.
JVM을한단계 더 거쳐 운영체제에 상관없이 실행.
-java 명령어란? javac 명령을 통해 컴파일한 .class 파일을 실행하는 명령어.
-javac 명령어란? .java 파일을 컴파일 하는 명령어. 

3.변수란 무엇인가

계속 변할 수 있음. 변수 하나에는 하나의 값을 저장.

4.변수 선언의 의미는?
메모리 공간의 할당, 변수의 이름과 용도가 결정. 해당 자료형의 크기만큼 메모리를 사용하는 
특정 변수를 지정하는 것.

5.들여쓰기를 왜 해야 하는가?
보기 편하기 위해서, 포함되어있는 코드를 알기 위해서, 서로간의 의사소통을 위해서.
프로그램 상에서는 아무런 의미가 없음.

6.주석은 컴파일시 어떻게 되는가?
코드 내용으로 인식되지 않고 연산 결과에 반영되지 않아 프로그램에 영향을 주지 않음.
