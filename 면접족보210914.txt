1. 자바의 명명 규칙에 대하여 설명하시오.
-클래스
 대문자로 시작. 길이제한 없음. 숫자로 시작할수 없음. 특수문자는 _ $ 만을 허용.
-메소드 변수
 소문자로 시작.

2. 아래의 접근 제한자에 대하여 설명하시오.
(1)private  (2)protected  (3)default  (4)public 
(1)-같은 클래스 내에서만 접근 가능 (2)-같은 패키지(폴더) 내 혹은 상속관계일 때 접근 가능
(3)-같은 패키지(폴더) 내에서 접근 가능 (4)-모두 접근 가능

3. 지역 변수에 접근 제한자를 붙이지 않는 이유는?
어차피 지역변수는 메소드 내에서 선언되기 때문에 해당 범위 내에서만 메모리를 할당.
게다가 접근 제한자를 통해 정보를 은닉하면 값을 참조할 수 없기 때문.

4. 캡슐화(encapsulation)에 대하여 설명하시오.
관련 변수와 함수를 하나의 클래스로 묶어 외부에서 쉽게 접근하지 못하도록 은닉하는 것.
객체에 대한 직접적인 접근을 막고, 객체가 제공하는 필드와 메소드를 통해서만 접근 가능.
코드를 좀 더 효율적이고 깔끔하게 정리할 수 있음.
