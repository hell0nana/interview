1. 자바의 명명 규칙에 대하여 설명하시오.
-클래스
 대문자로 시작. 길이제한 없음. 숫자로 시작할수 없음. 특수문자는 _ $ 만을 허용. 보통 명사.
-메소드
 소문자로 시작. 보통은 클래스의 기능을 표현하는 동사.
-변수
 소문자로 시작. 보통은 명사. _는 사용하지 않음.
-상수
 모두 대문자 사용. _로 단어 연결.
-패키지
 소문자 사용

2. 아래의 접근 제한자에 대하여 설명하시오.
(1)private  (2)protected  (3)default  (4)public 
(1)-동일 클래스 내에서만 접근 가능 (2)-동일 패키지(폴더) 내 혹은 상속관계일 때 접근 가능
(3)-동일 패키지(폴더) 내에서 접근 가능 (4)-모두 접근 가능(보통 한 개의 파일 안에는 한 개의 퍼블릭)

3. 지역 변수에 접근 제한자를 붙이지 않는 이유는?
지역변수는 메소드 내에서 선언되기 때문에 붙이지 않아도 해당 범위 내에서만 메모리를 할당.
또한 접근 제한자를 통해 정보를 은닉하면 값을 참조할 수 없음.

4. 캡슐화(encapsulation)에 대하여 설명하시오.
관련 변수와 함수를 하나의 클래스로 묶어 외부에서 쉽게 접근하지 못하도록 은닉하는 것.
객체에 대한 직접적인 접근을 막고, 객체가 제공하는 필드와 메소드를 통해서만 접근 가능.
데이터의 값을 유지할 수 있음. 코드를 좀 더 효율적이고 깔끔하게 정리할 수 있음.
오류의 범위를 최소화 함.
